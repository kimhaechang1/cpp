# 🛠️ C++ 컴파일 및 실행 가이드 (Windows 환경)

본 문서는 현재 워크스페이스(Windows OS, PowerShell 환경)에서 `g++` 컴파일러를 사용하여 C++ 코드를 빌드하고 실행하는 표준 명령어를 안내합니다.

## 1. 기본 컴파일 명령어 (C++17 표준)

가장 기본적으로 사용하는 C++17 표준 컴파일 명령어입니다.

```powershell
g++ -std=c++17 -o 출력파일명.exe 소스파일.cpp
```
- `-std=c++17`: C++17 최신 표준 문법을 사용하도록 지정합니다.
- `-o 출력파일명.exe`: 컴파일 결과물(실행 파일)의 이름을 지정합니다. (Windows에서는 `.exe` 필수)
- **예시**: `g++ -std=c++17 -o main.exe main.cpp`

## 2. 모던 C++ 컴파일 (C++20 / C++23 표준)

`std::expected`, `std::ranges`, `std::format` 등 최신 모던 C++ 문법을 실습할 때 사용하는 명령어입니다.

```powershell
# C++23을 요구하는 경우 (Module 09 이상)
g++ -std=c++23 -o 출력파일명.exe 소스파일.cpp

# C++20을 요구하는 경우
g++ -std=c++20 -o 출력파일명.exe 소스파일.cpp
```
- **예시**: `g++ -std=c++23 -o practice_cumulative.exe practice_cumulative_9_3.cpp`

## 3. 실행 명령어

성공적으로 빌드된 `.exe` 파일을 실행할 때는, PowerShell의 보안/경로 인식 정책상 **반드시 `./` 접두사**를 붙여야 합니다.

```powershell
./출력파일명.exe
```
- **예시**: `./main.exe`

---

## 🚨 주의사항 (Windows PowerShell 특화)

1. **명령어 동시 병합(`&&`) 주의**
   - Linux 환경이나 기존 명령 프롬프트(CMD)에서는 `g++ ... && ./main.exe` 처럼 한 줄로 즐겨 썼지만, **PowerShell에서는 `&&` 연산자가 설정이나 버전에 따라 예기치 않게 오작동**할 수 있습니다.
   - **안전 수칙**: 컴파일 명령어와 실행 명령어를 **반드시 두 줄에 나눠서 따로따로 입력**하세요.
     ```powershell
     # 올바른 방법
     g++ -std=c++23 -o main.exe main.cpp
     # (엔터 치고, 오류가 없으면)
     ./main.exe
     ```

2. **유니코드 및 이모티콘 에러 방지**
   - Windows 터미널의 `g++`(MinGW)은 소스 코드 내 특수 문자 처리에 민감합니다.
   - **소스 파일(`.cpp`, `.h`) 내부에는 주석이더라도 이모지(😄, 🚀 등)를 작성하지 마세요.**
   - 컴파일 인코딩 에러(`Unicode / ASCII error`)의 주범입니다. (마크다운 문서인 `.md` 파일에는 자유롭게 써도 됩니다.)

3. **IntelliSense(빨간 줄)와의 불일치**
   - 가끔 코드가 정상인데 VS Code 편집기에서 에러(빨간 줄)를 띄울 때가 있습니다. (특히 C++23 문법)
   - 터미널에서 `g++` 명령어로 컴파일했을 때 문제가 없다면, 그것은 에디터의 단순 구문 분석기 문제이므로 무시하고 진행하셔도 무방합니다.
