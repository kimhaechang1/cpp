# 정적 멤버(Static): "모두가 공유하는 단 하나의 기억"

## 1. 실무 프로젝트 상황: "지금 맵에 스폰된 몬스터는 총 몇 마리지?"

우리가 게임 서버나 클라이언트를 만들 때, **생성된 슬라임 객체가 총 몇 개**인지 세어야 하는 경우가 많습니다. (예를 들어, 일정 마릿수 이상 소환을 제한하기 위해)

당연히 슬라임 한 마리 구석에 속박된 개별 변수(`int hp`)로는 **전체 합계**를 알 수 없습니다.

가장 쉬운 방법은 전역 변수(Global Variable)를 쓰는 것입니다.
```cpp
int totalSlimeCount = 0; // 전역 변수

class Slime {
public:
    Slime() { totalSlimeCount++; } // 생성될 때마다 1 증가
    ~Slime() { totalSlimeCount--; } // 죽을 때마다 1 감소
};
```
하지만 **전역 변수는 만악의 근원**입니다. 코드 어느 곳에서든 `totalSlimeCount = 10000;` 처럼 멋대로 수정할 수 있고, 이름이 충돌하기도 쉽습니다. 분명히 '슬라임 전체 마릿수'는 슬라임(Slime)과 관련된 정보인데, 슬라임 클래스 밖에서 놀고 있다는 점도 객체지향 설계에 어긋납니다.

---

## 2. 해결책: 클래스에 속한 전역 변수, `static` 무리(멤버)

`static`(정적) 키워드를 멤버 변수 앞에 붙이면, 이 변수는 **객체(Instance)마다 생성되지 않고, 프로그램 전체에 오직 1개만 생성되며, 모든 객체가 이 변수 하나를 공유**하게 됩니다.

```cpp
class Slime {
private:
    int hp{100};             // 일반 멤버 변수 (슬라임마다 각자 1장소씩 가짐)
    static int totalCount;   // 정적 멤버 변수 (슬라임 100만마리가 있어도 오직 1개!)

public:
    Slime() {
        totalCount++; // 모든 슬라임이 하나의 변수에 숟가락을 얹음
    }
};

// C++ 특징: static 멤버 변수는 반드시 클래스 외부에서 물리적인(?) 메모리 공간을 잡아 할당해줘야 합니다.
int Slime::totalCount{0}; 
```

`static` 멤버는 클래스 이름에 종속되므로 접근할 때도 `슬라임1번.totalCount`가 아니라 `Slime::totalCount` 로 접근하는 것이 정석입니다.

---

## 3. 하드웨어 관점: 눈으로 보는 메모리 구조 차이 (제일 중요!)

`static` 변수는 스택(Stack)이나 힙(Heap)에 저장되지 않습니다. 프로그램이 실행될 때 로드되는 전역 데이터 영역(.data / .bss)이라는 전혀 다른 땅에 심어집니다!

실제로 주소를 찍어보면 확연히 다릅니다.

```cpp
Slime s1;
Slime s2;

// 일반 변수의 주소 : "서로 다른 객체니까 다른 메모리를 가짐"
std::cout << "&s1.hp : " << &s1.hp << std::endl;  // 결과: 0x0001
std::cout << "&s2.hp : " << &s2.hp << std::endl;  // 결과: 0x0005

// static 변수의 주소 : "두 객체가 똑같은 메모리를 쳐다보고 있음!"
std::cout << "&s1.totalCount : " << &s1.totalCount << std::endl; // 결과: 0xFFFF
std::cout << "&s2.totalCount : " << &s2.totalCount << std::endl; // 결과: 0xFFFF
```
*   `sizeof(s1)`을 출력하면 `hp`의 용량 4바이트만 뜰 뿐, `static`인 `totalCount` 용량은 합산되지 않습니다. 객체 몸통 안에 들어있지 않기 때문입니다!

## 4. 정적 멤버 함수 (Static Member Function)

변수뿐 아니라 함수에도 `static`을 붙일 수 있습니다.
`static` 멤버 함수는 **"객체가 하나도 생성되지 않아도 호출할 수 있는 함수"**입니다.

```cpp
class Slime {
private:
    static int totalCount;
public:
    // 일반 함수: 슬라임이 태어나야 공격할 수 있음. p.Attack()
    void Attack() { ... } 
    
    // static 함수: 슬라임이 0마리여도 호출 가능! Slime::GetCount()
    static int GetCount() { return totalCount; }
};
```

### 🚨 초보자들이 가장 많이 하는 실수 (컴파일 에러!)
`static` 함수 안에서는 `this` 포인터가 존재할 수 없습니다. (객체가 없어도 호출될 수 있게 만들었으니까요!)
따라서 **`static` 함수 내부에서는 `static` 변수만 접근 가능하며, 일반 `hp`같은 멤버 변수는 절대 건드릴 수 없습니다.** (누구의 hp인지 알 길이 없으므로)

### 👉 [생각해보기]
우리가 게임에서 객체별로 **고유 번호(Unique ID)**를 부여하고 싶습니다 (슬라임1, 슬라임2, 슬라임3...).
각 객체의 멤버 변수 `int myId;`에 매번 새로운 고유 번호를 세팅하려면, 방금 배운 `static` 변수를 어떻게 응용하면 좋을까요?

---

### 🗨️ 회창님의 생각과 피드백 (Q&A)

**Q. 회창님의 답변:**
> "슬라임1, 슬라임2... 이런 식이라면 "슬라임"이라는 문자열 정도는 static에 둘 순 있겠다는 생각이 드네. 그리고 글로벌 카운팅 방식으로 ID를 세팅하면 될 것 같아."

**A. 멘토의 피드백:**
정확한 설계적 직관입니다! 

1. **상수화된 공용 데이터 분리 (`static const string`)**
   "슬라임"이라는 공통된 접두사(Prefix 문자열)는 모든 객체가 똑같이 가지는 값이므로, 메모리 낭비를 줄이기 위해 `static`으로 선언하여 객체 간 공유하는 것은 훌륭한 최적화 아이디어입니다.

2. **고유 번호(Unique ID) 발급기 패턴 (`글로벌 카운터`)**
   "내가 몇 번째로 태어났는지" 알기 위해서는 이전에 태어난 녀석이 몇 번이었는지 기억하는 공용 카운터가 필요합니다. 
   이를 위해 (1) 클래스의 총 누적 생성 횟수를 세는 `static` 변수를 두고, (2) 생성자가 호출될 때마다 1 올린 뒤 그 값을 자신만의 방(`myId`)에 복사해 넣습니다.

```cpp
class Slime {
private:
    static int globalCounter; // 생성된 모오오든 슬라임의 '누적' 마릿수
    int myId;                 // 나만을 위한 고유 민증 번호

public:
    Slime() {
        globalCounter++;            // 공용 변수를 1 올림 (ex: 0 -> 1)
        this->myId = globalCounter; // 내 고유 번호에 '1'을 새김!
    }
    
    void PrintMyName() {
        // 회창님 아이디어처럼 "슬라임"이라는 규칙과 조합!
        std::cout << "나는 슬라임" << myId << " 이다!" << std::endl;
    }
};

// 외부 초기화 (반드시 필요함)
int Slime::globalCounter = 0; 
```

이 "고유 번호(ID) 자동 발급" 패턴은 게임의 네트워크 서버에서 **각 유저(세션)나 아이템에 겹치지 않는 ID 번호를 매길 때 100% 똑같이 쓰이는 초핵심 기법**입니다!

**Q. 회창님의 추가 질문:**
> "static const 로 선언한다는 건 정적인 변수 선언과 동시에 수정이 불가능한 거잖아. 그러면 반드시 초기화되어 있어야 컴파일 에러가 발생하지 않는 건가?"

**A. 멘토의 피드백:**
아주 날카로운 지적입니다! `const`(상수) 키워드가 붙었기 때문에 **"태어나는 그 순간에 값이 정해져야 하고, 이후에는 절대 바꿀 수 없다"**는 강력한 규칙이 생깁니다. 따라서 반드시 초기화가 되어야만 합니다.

하지만 C++에서는 타입(자료형)에 따라 초기화하는 "장소"가 다릅니다.

1. **`int`, `char`, `bool` (기본 정수형 계열)**
   가장 가벼운 타입들이라 클래스 선언 내부에서 직접 `=`을 써서 초기화하는 것을 유일하게 허락합니다.
   ```cpp
   class Slime {
   public:
       static const int MAX_LEVEL = 99; // 내부 직접 초기화 성공!
   };
   ```

2. **`std::string`, `float`, 클래스 객체 (그 외 모든 것)**
   "슬라임"이라는 접두사를 쓰기 위해 `std::string`을 `static const`로 선언하면, 내부 초기화 특혜를 주지 않습니다. 반드시 **일반 static 변수처럼 외부에서 초기화**해야 합니다.
   ```cpp
   class Slime {
   public:
       static const std::string PREFIX; // 내부에서는 선언만!
   };
   // 외부에서 초기화하면서 "이건 값이 영원히 이거야!" 라고 못 박음
   const std::string Slime::PREFIX = "슬라임"; 
   ```

3. **✨ Modern C++ (C++17 이상)의 마법 해결책: `inline` 키워드**
   외부로 끄집어내서 초기화하는 건 솔직히 귀찮고 가독성도 떨어집니다. 그래서 C++17부터는 `inline` 키워드를 추가하여, 자료형에 상관없이 **클래스 내부에서 즉시 초기화가 완벽하게 허용**되도록 개선되었습니다!
   ```cpp
   class Slime {
   public:
       // C++17 부터 가능: inline을 붙이면 string도 클래스 내부에서 깔끔하게 초기화 종료!
       static inline const std::string PREFIX = "슈퍼_슬라임_"; 
   };
   ```
   "static이고 const니까 값부터 넣고 봐야지!" 라는 회창님의 직관은 완벽하게 옳았습니다! 옛날 규칙들이 살짝 꼬여있었을 뿐이죠.
