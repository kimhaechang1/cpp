# Module 07: 순수 가상 함수와 추상 클래스 (Abstract Class)

## 🎮 실무 프로젝트 상황: "너는 설계도일 뿐이야"
게임 엔진을 설계하다 보면, 다양한 클래스들의 '부모' 역할을 하지만 그 자체로는 절대 존재해서는 안 되는 클래스들이 있습니다.

- **`Unit`**: 모든 유닛의 부모지만, 맵 위에 '그냥 유닛'이 소환될 수는 없습니다. (오크, 슬라임, 플레이어 중 하나여야 함)
- **`Shape`**: 모든 도형의 부모지만, '그냥 도형'을 그릴 수는 없습니다. (삼각형, 사각형 등 구체적인 형태가 필요함)
- **`Weapon`**: 모든 무기의 부모지만, 공격력이 없는 '그냥 무기'는 존재하지 않습니다.

이런 클래스들은 자식 클래스가 반드시 구현해야 할 **"표준 인터페이스(규칙)"**만 정의하고, 자신은 실체화(Instance 생성)되지 못하도록 막아야 합니다. 이것이 바로 **추상 클래스**를 만드는 이유입니다.

---

## 💡 이론: 순수 가상 함수 (Pure Virtual Function)

가상 함수 선언 뒤에 **`= 0;`**을 붙이면, 이 함수는 **"순수 가상 함수"**가 됩니다.

```cpp
class Unit {
public:
    // "공격은 반드시 해야 하는데, 어떻게 할지는 자식이 정해라!"
    virtual void Attack() = 0; 
};
```

### 🧠 순수 가상 함수의 특징
1.  **본체가 없음**: 함수 선언만 있고 정의(`{ }`)가 없습니다.
2.  **강제성**: 이 클래스를 상속받는 자식 클래스는 **반드시** 이 함수를 오버라이딩해야 합니다. 구현하지 않으면 자식 클래스도 추상 클래스가 되어 인스턴스를 만들 수 없습니다.

---

## 🏗️ 추상 클래스 (Abstract Class)

하나 이상의 순수 가상 함수를 포함하는 클래스를 **추상 클래스**라고 부릅니다.

### 🚨 추상 클래스의 제약 사항
추상 클래스는 설계도일 뿐이므로, **객체(Instance)를 생성할 수 없습니다.**

```cpp
Unit myUnit; // ❌ 컴파일 에러! (추상 클래스는 실체화할 수 없음)
Unit* pUnit = new Orc(); // ✅ 가능! (부모 포인터로 자식 객체를 가리키는 다형성은 유지됨)
```

---

## 🔍 하드웨어 및 컴파일 레벨 검증
추상 클래스가 정말로 인스턴스화되지 않는지, 그리고 왜 필요한지를 코드로 확인해 봅시다.

### 1. 인스턴스화 금지 확인
컴파일러는 추상 클래스 타입의 변수를 발견하면 즉시 에러를 냅니다. 이는 런타임에 발생할 수 있는 "구현되지 않은 함수 호출" 사고를 컴파일 타임에 완벽히 차단하는 안전 장치입니다.

### 2. 메모리 상의 vtable
추상 클래스의 vtable에는 순수 가상 함수 자리에 `0` (NULL) 혹은 에러 처리 함수 주소가 들어갑니다. 자식 클래스가 이를 오버라이딩하면 vtable의 해당 칸이 실제 자식 함수의 주소로 채워지게 됩니다.

---

## 🛠️ 인터페이스(Interface) 설계 패턴
C#이나 Java에는 `interface`라는 키워드가 따로 있지만, C++에서는 **오직 순수 가상 함수로만 이루어진 클래스**를 인터페이스로 사용합니다.

```cpp
class IAttacker { // Interface 역할을 하는 추상 클래스 (보통 이름 앞에 I를 붙임)
public:
    virtual ~IAttacker() {} // 가상 소멸자는 필수!
    virtual void TakeDamage(int amount) = 0;
    virtual void UseSkill() = 0;
};
```

이렇게 설계하면, `IAttacker` 포인터 배열을 만들어 서로 다른 유닛들이 각자의 방식대로 공격하게 만드는 **강력한 다형성**을 구현할 수 있습니다.

## ❓ 궁금한 점이 생기셨나요?

### Q. 순수 가상 함수도 본체(`{ }`)를 가질 수 있나요? (Java의 default 메소드처럼)
**A. 네, 가능합니다!** 
하지만 Java와는 성격이 조금 다릅니다. C++에서는 `= 0;`으로 선언된 함수라도 클래스 외부에서 본체를 정의할 수 있습니다.

- **본체 위치**: 반드시 **클래스 정의 밖**에서 정의해야 합니다. (클래스 내부 `{ }` 작성은 문법 에러)
- **차이점**: 본체가 있더라도 자식 클래스는 이 함수를 **반드시 오버라이딩**해야 합니다.
- **용도**: 자식 클래스가 자신의 오버라이딩 함수 내에서 `Parent::Function()` 형태로 부모의 공통 로직을 명시적으로 호출하고자 할 때 사용합니다.

### Q. 본체가 없는 순수 가상 함수를 호출하면 어떻게 되나요?
**A. 컴파일러는 가차 없습니다.**
- 자식 클래스에서 `Base::Function();` 처럼 명시적으로 호출했는데 부모 쪽에 본체가 없다면 **링크 에러(Linker Error)**가 발생합니다.
- 부모 클래스의 **생성자/소멸자**에서 순수 가상 함수를 호출하려고 시도하면 **`pure virtual function call`** 이라는 치명적인 런타임 에러가 발생하며 프로그램이 강제 종료됩니다. (객체가 아직 생성/소모 중이라 가상 함수 테이블이 불안정하기 때문입니다.)

### Q. 추상 클래스에 일반 멤버 변수를 넣어도 되나요?
**A. 네, 물론입니다.** 추상 클래스도 멤버 변수, 멤버 함수를 모두 가질 수 있습니다. 단지 '독립적으로 인스턴스화'만 불가능할 뿐입니다.
