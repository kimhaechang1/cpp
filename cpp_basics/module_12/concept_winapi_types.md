# 심화 학습: Windows API 자료형과 비트 연산의 이해

네트워크 프로그래밍의 문턱에서 만나는 생소한 자료형들과 비트 연산 매커니즘을 정리합니다. 이 지식은 단순히 Winsock뿐만 아니라, 저수준 시스템 프로그래밍 전반에 걸쳐 사용되는 핵심 원리입니다.

---

## 1. Windows API의 독특한 자료형 (헝가리안 표기법)

현대 C++ 개발자에게 익숙하지 않은 이 이름들은 과거 16비트 메모리 모델 시절의 유물이나, 현재까지도 Windows 표준 인터페이스로 사용되고 있습니다.

- **`WORD` (2 Bytes)**: `unsigned short`와 동일합니다. 16비트(2바이트) 정수를 의미합니다.
- **`DWORD` (4 Bytes)**: **D**ouble **Word**의 약자로, `unsigned int`와 동일한 32비트 정수입니다.
- **`LP` (Long Pointer)**: 포인터(`*`)를 의미하는 접두사입니다.
    - 예: `LPWSADATA` = `WSADATA` 구조체에 대한 포인터 (`WSADATA*`)
- **`BYTE` (1 Byte)**: `unsigned char`와 동일한 8비트 정수입니다.

---

## 2. MAKEWORD 매크로와 버전 패킹(Packing)

`WSAStartup(MAKEWORD(2, 2), ...)`에서 사용하는 `MAKEWORD`는 두 기의 8비트 숫자를 하나의 16비트 `WORD`로 합치는 역할을 합니다.

### 💡 매크로 동작 원리
```cpp
#define MAKEWORD(low, high) ((WORD)(((BYTE)(low)) | ((WORD)((BYTE)(high))) << 8))
```
1.  **Low Byte**: 첫 번째 인자를 하위 8비트에 배치합니다.
2.  **High Byte**: 두 번째 인자를 8비트 왼쪽으로 밀어서(`<< 8`) 상위 비트에 배치합니다.
3.  **OR (|)**: 두 비트를 합쳐서 하나의 16비트 상자를 완성합니다.

**결과**: `MAKEWORD(2, 2)` -> `[0000 0010]` + `[0000 0010]` -> `0000 0010 0000 0010` (**0x0202**, 십진수 **514**)

---

## 3. 16진수 0xFF와 비트 마스크(Bit Mask)

데이터를 추출할 때 사용하는 `0xFF`는 비트 연산의 '필터' 역할을 합니다.

### 💎 왜 하필 0xFF 인가?
- **16진수 F** = 십진수 15 = 이진수 **`1111`** (4비트의 전원을 모두 켠 상태)
- **0xFF** = **`1111 1111`** (8비트 즉, 1바이트 전체를 모두 1로 가득 채운 상태)

### 🗡️ 데이터 추출(Extracting) 기법
16비트 데이터(`0x0202`)에서 각 버전을 다시 뽑아내는 방법입니다.

1.  **하위 바이트 추출 (Major Version)**:
    - `0x0202 & 0x00FF`
    - `0xFF`와 `&` 연산을 하면 '1'인 부분의 데이터만 통과하고 나머지는 0으로 지워집니다.
2.  **상위 바이트 추출 (Minor Version)**:
    - `(0x0202 >> 8) & 0xFF`
    - 오른쪽으로 8칸을 밀어서(`>> 8`) 상위 비트를 하위 비트 자리로 내린 뒤, 똑같이 `0xFF` 필터를 적용합니다.

---

## 🏗️ 요약 및 결론
- **WORD**는 16개의 비트로 이루어진 상자입니다.
- 이 상자를 절반으로 나눠서 **Major**와 **Minor** 버전 정보를 담습니다.
- **비트 시프트(`<<`, `>>`)**는 데이터를 옮기는 손이고, **비트 마스크(`& 0xFF`)**는 원하는 데이터만 걸러내는 체(Sieve)입니다.

이 원리를 이해했다는 것은, 데이터가 메모리 상에서 어떻게 배열되고 조작되는지를 볼 수 있는 **'시스템 프로그래머의 눈'**을 갖게 되었다는 것을 의미합니다.
