# MODULE 02: 제어 흐름 (Control Flow)

## 1. 개요
이미 알고리즘 문제 풀이를 통해 `if`문과 `for`문은 상당히 익숙해지셨을 겁니다.
이번 모듈에서는 **"게임 개발"**에서 특히 자주 쓰이는 제어문(`switch`, `while`)과 흐름 제어(`break`, `continue`)를 집중적으로 다룹니다.

---

## 2. 분기문: switch vs if

### 2.1. switch문의 특징
- `if-else if`가 너무 길어질 때 사용하면 가독성이 좋아집니다.
- **정수(int)나 열거형(enum)** 판별에 최적화되어 있습니다. (문자열 `string`이나 범위 `10 < x < 20` 판정 불가)
- **게임 활용**: 캐릭터의 상태(IDLE, RUN, ATTACK)나 아이템 타입을 구분할 때 **국룰**처럼 쓰입니다.

```cpp
// Enum과 Switch는 영혼의 단짝입니다.
switch (playerState) {
    case State::IDLE:
        // 숨쉬기 애니메이션 재생
        break; // 까먹으면 밑으로 뚫고 내려갑니다! (Fall-through)
    case State::ATTACK:
        // 공격 데미지 계산
        break;
    default:
        // 예외 처리
        break;
}
```

### 2.2. 언제 무엇을 쓸까?
- **범위 체크** (점수가 90점 이상): `if`
- **복합 조건** (A이고 B이거나 C): `if`
- **딱 떨어지는 상태 값** (직업 = 전사): `switch`

## 3. 반복문: while vs for

### 3.1. while문
- **"조건이 참인 동안"** 계속 반복합니다.
- 몇 번 반복할지 모를 때 주로 씁니다.
- **게임 활용**: **"게임 루프"** (게임이 꺼지기 전까지 무한 반복)

```cpp
while (hp > 0) {
    // 플레이어가 살아있는 동안 계속 전투
    Attack();
}
```

### 3.2. do-while문
- 무조건 **"최소 한 번"**은 실행하고 조건을 검사합니다.
- **활용**: 입력을 검증할 때 ("제대로 된 값을 입력할 때까지 반복")

```cpp
do {
    cout << "1~3 사이 메뉴를 선택하세요: ";
    cin >> input;
} while (input < 1 || input > 3);
```

## 4. 흐름 끊기: break & continue

- **break**: 반복문을 즉시 **탈출**합니다. ("그만해!")
  - 예: 아이템을 찾으면 탐색 종료
- **continue**: 아래 코드를 무시하고 **다음 반복**으로 넘어갑니다. ("이번만 패스!")
  - 예: 짝수만 출력하기, 죽은 몬스터는 때리기 건너뛰기

```cpp
for (Monster& m : monsters) {
    if (m.IsDead()) continue; // 시체는 때리지 말고 패스
    
    m.TakeDamage(10);
    
    if (m.IsBoss() && m.IsDead()) {
        cout << "게임 클리어!";
        break; // 보스 잡으면 루프 완전 종료
    }
}
```

## 6. [Q&A] Range-based `for`문과 `auto`의 치명적인 함정

**Q1. `for (auto cell : resources)` 에서 `&`를 빼면 왜 컴파일 에러가 나거나 성능이 박살나나요?**
*   **A**: C++에서 `auto`는 아주 무서운 기본 규칙을 가집니다. **"기본적으로 `auto`는 기존의 참조(`&`)나 상수 속성(`const`)을 전부 강제로 벗겨내고 무조건 '새로운 복사본'을 만들어버린다!"**
*   `&` 없이 순회를 돌면, 배열에서 뽑아낸 원소를 매번 '새로 복사'해서 변수에 집어넣으려고 시도합니다.
*   만약 원소가 무거운 클래스 객체이거나, 복사가 아예 금지된 `std::unique_ptr` 였다면 당장 메모리가 낭비되거나 컴파일 대폭발(`use of deleted function`)이 일어납니다.

**[✅ 해결책 - C++의 절대 관례 (국룰)]**
복사를 방지하고 원본의 주소만 안전하게 쳐다보려면, 제어문에서 순회를 돌 때 귀찮더라도 **`const auto&`** 를 사용하는 것을 무조건적인 습관으로 들여야 합니다!
```cpp
for (const auto& cell : this->resources) { 
    // "복사본 안 만들고, 원본 요소의 주소에 이름표(cell)만 달아서 읽어올게!"
}
```

> 👉 **[심화 개념 연결]**: 위에서 언급한 "복사가 아예 금지된 `std::unique_ptr`"가 대체 무엇인지 궁금하신가요? 왜 모던 C++은 똑똑한 포인터를 만들고 함부로 복사를 막아두었는지 그 철학이 궁금하시다면, 언제든 **[Module 09: 스마트 포인터](../module_09/concept_smart_pointers.md)** 문서를 먼저 엿보셔도 좋습니다!
