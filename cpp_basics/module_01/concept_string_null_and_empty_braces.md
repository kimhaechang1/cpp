# Module 1 보충: 빈 중괄호 `{}`의 비밀과 std::string의 NULL 초기화

## 1. Uniform Initialization의 빈 중괄호 `{}`는 무슨 값을 넣을까?
기본값을 생략하고 냅다 빈 중괄호 `{}`만 붙여놓으면, C++ 컴파일러는 해당 타입을 **가장 안전한 기본값(Zero-initialization 또는 Default Constructor)**으로 알아서 아주 예쁘게 초기화해 줍니다. 

*   `int a{};` ➡️ `0`이 들어갑니다.
*   `float b{};` ➡️ `0.0f`가 들어갑니다.
*   `bool c{};` ➡️ `false`가 들어갑니다.
*   `int* ptr{};` ➡️ `nullptr`가 들어갑니다.
*   `std::string s{};` ➡️ **빈 문자열 `""`**이 들어갑니다. (생성자 내부적으로 글자 수가 0개인 가장 정상적이고 안전한 상태로 자체 세팅됨)

전통적인 C 언어 스타일에서 변수를 선언만 하고 값을 넣지 않으면 과거의 흔적인 **쓰레기값(Garbage Value)**이 들어가서 치명적인 버그를 일으킵니다. 하지만 C++11부터는 조건 없이 무조건 변수 뒤에 `{}`를 붙여주는 습관만 들여도 초기화 누락 버그를 100% 원천 차단할 수 있는 기적의 문법입니다!

## 2. 왜 `std::string s = nullptr;`는 프로그램이 터질까?
타 언어(C#, Java)와 달리 C++의 `std::string`은 단일 포인터가 아니라, 문자열의 길이와 실제 메모리 주소를 스스로 안전하게 관리해 주는 '똑똑한 래퍼 객체(Object)'입니다.
*   **오류 원인**: 만약 `std::string` 객체의 생성자에 `nullptr`를 억지로 구겨 넣으면, 내부 로직은 넘겨받은 메모리 주소(0번지, 접근 금지 구역)로 직접 뛰어가서 글자 수를 세려고(`strlen`) 시도합니다. 0번지 접근은 운영체제가 절대 허락하지 않는 해킹 및 오류 행위이므로, 즉시 프로세스(Crash)를 강제 종료시켜 버립니다.
