# Antigravity 폴더 구조 및 역할 가이드

이 문서는 **Antigravity 시스템(Agentic Coding)**에서 사용하는 특수 폴더들과, 우리 프로젝트(`GEMINI.md`)에서 정의한 커스텀 폴더들의 역할과 사용법을 설명합니다.

---

## 1. ⚙️ 시스템 정의 폴더 (System-Defined)
Antigravity 시스템이 **"기능적으로 인식하고 작동"**시키는 특수 폴더들입니다.
이 폴더들의 이름과 위치는 시스템에 의해 고정되어 있습니다.

### 📂 `.agent/workflows/` (가장 중요 ⭐)
- **기능**: **[Slash Command]** 등록소
- **핵심 역할**: 이 폴더에 `.md` 파일을 만들면, 채팅창에서 `/파일명` 명령어로 즉시 호출할 수 있습니다.
- **작동 원리**:
    - `basic.md` 생성 -> 채팅창에 `/basic` 입력 가능해짐.
    - 파일 내용은 에이전트가 수행할 **"절차(Procedure)"**를 정의합니다.
- **예시**:
    ```markdown
    1. Read .agent/roles/basic_tutor.md
    2. Adopt Persona
    3. Start Session
    ```

### 📂 `.gemini/artifacts/`
- **기능**: **[Brain Storage]**
- **핵심 역할**: 에이전트가 생성한 `task.md`(할 일 목록), `implementation_plan.md`(구현 계획) 등이 저장되는 곳입니다.
- **작동 원리**: 사용자가 직접 수정하기보다, 에이전트가 생각하고 계획을 수립하는 **"작업 기억 공간"**으로 쓰입니다.
- **주의**: 이 폴더를 지우면 에이전트가 현재 진행 중인 작업 맥락을 잃어버릴 수 있습니다.

### 📂 `.agent/skills/` (고급)
- **기능**: **[Skill Library]**
- **핵심 역할**: 특정 전문 작업(예: 웹 크롤링, 데이터 분석)을 수행하는 재사용 가능한 기술 모음입니다.
- **작동 원리**: 현재 우리 프로젝트에서는 사용하지 않지만, 나중에 "복잡한 기능을 모듈화"할 때 사용할 수 있습니다.

---

## 2. 🤝 사용자 정의 폴더 (User-Defined Convention)
Antigravity 시스템이 강제하는 것이 아니라, **우리 프로젝트(`GEMINI.md`)의 아키텍처 규칙**에 따라 약속한 폴더들입니다.
폴더명을 바꿔도 시스템 오류는 안 나지만, 워크플로우가 깨질 수 있습니다.

### 📂 `.agent/roles/`
- **기능**: **[Persona Definition]**
- **핵심 역할**: "누가 가르칠 것인가?"를 정의하는 페르소나 파일(`basic_tutor.md` 등) 보관소입니다.
- **왜 쓰는가?**: 워크플로우(`workflows/*.md`)에서 "이 파일을 읽어라"라고 명시했기 때문에 작동하는 것입니다.
- **규칙**: 이곳의 파일들은 **행동 지침(Behavior)**과 **교육 철학**만 담고, 구체적인 커리큘럼 데이터는 담지 않습니다.

### 📂 `docs/curriculum/`
- **기능**: **[Database]**
- **핵심 역할**: "무엇을 배울 것인가?"를 정의하는 커리큘럼 파일(`CPP_BASICS_CURRICULUM.md` 등) 보관소입니다.
- **규칙**: 에이전트는 이곳의 파일을 **"데이터베이스"**처럼 읽어서 진행 상황을 체크(✔)합니다.

### 📄 `GEMINI.md`
- **기능**: **[Constitution]**
- **핵심 역할**: 이 프로젝트의 **"헌법"**입니다.
- **규칙**: 모든 에이전트(`roles`)와 워크플로우(`workflows`)는 이 파일에 정의된 **"시스템 아키텍처 원칙"**을 최우선으로 따릅니다.
